def zillow_annual_growth(df):
    '''
    This function takes the last 10 years of Zillow data and returns a dataframe with average year-over-year growth of
    2br real estate value by zip code.
    
    This function was created because the Zillow data is from 6/2017, while the AirBnb data is from 7/2019.
    I needed a way to estimate the real estate value for 2019, approximately 2 years later.
    
    If I know the average annual growth of real estate value by zip code, I can estimate what the real estate
    value would be in 2019, which will align with the AirBnb data.
    '''
    
    zip_dict = {}
    
    ## Taking the real estate value annually (once every 12 months)
    for row in range(len(df)):
        costs = []
        rolling_annual_growth = []
        
        for i in last_10_yrs:
            costs.append(df[i][row])
            
            if len(costs) % 12 == 0:
                rolling_annual_growth.append(np.mean(costs[-12:]))
                zip_dict.update({df['RegionName'][row] : rolling_annual_growth})
            else:
                continue

    rolling_growth_dict = {}
    
    ## Calculating real estate growth from the last 10 years
    for key in zip_dict.keys():
        avg_growth = []
        for num in range(len(zip_dict[key])):
            if num != 0:
                avg_growth.append((zip_dict[key][num] - zip_dict[key][num-1])/zip_dict[key][num-1])
        rolling_growth_dict.update({key : avg_growth})

    growth_dict = {}
    
    ## Averaging the 10 year growth numbers into one average annual growth
    for key in rolling_growth_dict.keys():
        avg_growth = np.mean(rolling_growth_dict[key])
        growth_dict.update({key : avg_growth})

    growth_df = pd.DataFrame.from_dict(growth_dict, orient = 'index').reset_index()
    growth_df = growth_df.rename(columns = {'index' : 'zip', 0 : 'growth'})
    growth_df['zip'] = growth_df['zip'].astype(str)
    
    return growth_df

## Filtering the Zillow data for only NYC
ny_br_df = br_df[br_df['RegionName'].isin(ny_zip_list)].reset_index(drop = True)

## Calculating average annual growth for NYC Zillow data
growth_df = zillow_annual_growth(ny_br_df)

## Calculating median real estate value by zip code
zip_cost = ny_br_df.groupby('RegionName')['2017-06'].agg('median').sort_values(ascending = False).reset_index()
zip_cost['RegionName'] = zip_cost['RegionName'].astype(str)

## Filtering the AirBnb data for only NYC; converting median price by zip code to annual income, assuming 75% occupancy
ny_listings = listings[listings['zipcode'].isin(ny_zip_list)].groupby('zipcode')['price'].agg(['count','median']).reset_index()
ny_listings['annual_income'] = (ny_listings['median']*0.75*365.25).round(2)
ny_listings = ny_listings.rename(columns = {'count' : 'AirBnb listings by zipcode'})

## Merging data sources together to create master dataframe that contains
## cost data, income data, and, created below, profit data
profit_df = pd.merge(growth_df, pd.merge(ny_listings[['zipcode','AirBnb listings by zipcode','annual_income']], zip_cost, 
                    left_on = 'zipcode', right_on = 'RegionName'), left_on = 'zip', right_on = 'zipcode')
del profit_df['RegionName']
del profit_df['zip']

## Converting Zillow real estate value to 2019 values using the difference in dates between data sources
## This will work for future data being added, but column names will need to be adjusted in the future
profit_df['2019_cost_est'] = pow((1 + profit_df['growth']), year_dif)*profit_df['2017-06']

## Calculating various levels of profit numbers
profit_df['1yr_profit'] = profit_df['annual_income'] - profit_df['2019_cost_est']
profit_df['5yr_profit'] = (profit_df['annual_income'] * 5) - profit_df['2019_cost_est']
profit_df['10yr_profit'] = (profit_df['annual_income'] * 10) - profit_df['2019_cost_est']
profit_df['30yr_profit'] = (profit_df['annual_income'] * 30) - profit_df['2019_cost_est']
profit_df['50yr_profit'] = (profit_df['annual_income'] * 50) - profit_df['2019_cost_est']
profit_df['zipcode'] = profit_df['zipcode'].astype(int).astype(str)

Conclusion
¶

The zip codes that are best to invest in depend on the timeframe that the investor is looking for. In this assignment, the buyer is purchasing the property with cash up front. This means the buyer is taking a huge loss in the beginning, but every year after that works to recoup his loss.

In the short term (1-10 years), it's best to invest in low-cost properties that have little AirBnb competition (Staten Island, as seen in attached graph). This is because the 'huge loss in the beginning' is much less huge. The buyer has a much smaller hole to dig him/herself out of, and can do so much more quickly.

In the long term (30-50 years), it would be best to invest in the properties that generate the most income (Manhattan). This is because, though there would be a huge loss at the beginning, every year after that is considered profit. It takes longer to dig him/herself out of the hole, but once this happens, the profits begin to stack up very quickly.

4. What's Next
¶

In a future analysis, I would like to work with a more complete set of AirBnb zip codes. I ended up having only 25 zip codes that were available in all datasets, and this is why we see empty, white zip codes on the NYC map. It would be great to have the full picture of NYC available.

I would also like to have enhanced Zillow data that could break down property values by more factors (square footage, amenities, etc.). This would allow me to use much more data from the AirBnb dataset that I'm currently not able to use. For example, I might have found that 500 sq. foot Manhattan properties cost 2x as much as 2000 sq. foot Staten Island properties, but they can be rented out for 5x more. This would have enabled a layer of analysis that would be very interesting. I could have also found that properties with new furniture sell for 20% more, despite only costing 5% more.

All in all, I had a lot of fun working on this project. I hope you enjoyed reading through my analysis - I look forward to discussing next steps with the Capital One team!
